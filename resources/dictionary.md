# 단어집 📓
| **용어** | **정의** |
| --- | --- |
| **Throughput (처리량)** | 특정 기간 내에 처리된 쿼리 또는 요청의 수 (처리량) |
| **Latency (레이턴시)** | 요청이 시작된 후 응답이 돌아오기까지 걸리는 순수 처리 지연 시간 |
| **Response Time (응답 시간)** | 레이턴시에 네트워크 지연, 대기열 지연 등 기타 요소를 모두 포함한 최종 시간 |
| **Off-the-shelf (기성)** | 기성품이라는 뜻으로, 커스터마이징 없이 바로 사용 가능한 라이브러리/모델/솔루션 |
| **Ticket (고객 지원 티켓)** | 고객이 회사에 문의하거나 문제를 제기할 때 생성되는 문서/요청서 같은 것. 영화관 티켓처럼 번호표를 받는다는 의미에서 유래  |
| **Use Case (유스 케이스)** | 어떤 기술을 ‘어디에, 어떻게 쓰는지’를 나타내는 구체적인 사용 시나리오. ML 유스 케이스의 경우 머신러닝을 써서 해결하는 구체적인 문제/사용 시나리오를 뜻함 |
| **First Party Data (퍼스트 파티 데이터)** | 회사에서 사용자 또는 고객에 대해 이미 수집하고 있는 데이터 |
| **Second Party Data (세컨드 파티 데이터)** | 다른 회사에서 자체 고객에 대해 수집하는 데이터. 제공받으려면 비용을 지불해야 함 |
| **Third Party Data (서드 파티 데이터)** | 직접적인 고객이 아닌 공공 데이터를 수집한 것 |
| **Data Serialization (데이터 직렬화)** | 데이터 구조나 객체 상태를 저장 혹은 전송하고 나중에 재구성할 수 있는 포맷으로 변환하는 프로세스. 컴퓨터/네트워크로 전송하려면 데이터가 연속된 일련의 바이트 형태로 구성되어야 하기 때문에 필요한 처리 |
| **Text Blob (텍스트 블롭)** | DB나 파일에 문자열로 저장되는 비정형 또는 포맷이 지정되지 않은 텍스트 데이터 덩어리. 구조적이지 않아 처리나 분석하기 어려운 대량의 텍스트 데이터 |
| **Turing-complete (튜링 완전)** | SQL로 모든 계산 문제를 해결할 수 있다는 의미 (시간, 메모리 고려 없이) |
| **Schema-on-read (스키마-온-리드)** | 문서 DB에 데이터를 기록할 때는 스키마를 준수해야 하는 제약 조건이 없지만, 데이터를 읽는 애플리케이션은 스키마를 맞추기 위한 처리 작업이 필요함 |
| **Locality (지역성)** | 동일한 영역 내에서 관련 데이터를 저장하도록 하는 원칙 |
| **Bytestring (바이트스트링)** | 문자가 아니라 바이트(0-255 숫자)들의 나열로 이루어진 데이터 덩어리. 사람이 읽는 문자열이 아니라 컴퓨터가 다루는 원시 데이터 |
| **Opt-in (옵트인)** | 사용자가 명시적으로 동의해야만 데이터 수집이 가능한 방식 |
