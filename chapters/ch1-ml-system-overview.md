### 1.1 머신러닝을 사용해야 하는 경우 (p. 35)

- 머신러닝이란?
    
    : 기존 데이터로부터 복잡한 패턴을 학습하고 미지의 데이터에 대해 예측을 수행하는 접근법
    
- ML 적용을 위한 5가지 핵심 조건 **🔑**
    - 학습 능력: 시스템이 명시적 규칙이 아닌 **데이터로부터 관계를 스스로 학습**할 수 있어야 함
        
        → 보험 사례: 의료기관 청구 패턴을 입력으로 사기 가능성을 예측하는 모델 학습
        
    - 복잡한 패턴: 조회 테이블, 규칙 기반 시스템 **이상의 복잡도를 지녀야** ML 적용이 유의미. 단 패턴이 없는 무작위 현상의 경우도 마찬가지로 부적합
        
        → 보험 사례: 둔갑치료, 영수증 쪼개기 등 다양한 사기 유형의 복합적 패턴 탐지
        
    - 데이터 가용성: 사용 가능한 데이터가 있거나 수집 가능해야 함
        
        →  보험 사례: 장기보상팀의 수동 조사 결과를 축적하여 사기 패턴 데이터베이스 구축
        
    - 예측 문제: 문제가 **예측 형태로 재구성 가능**해야 ML 적용이 유효 (미래뿐 아니라 현재/과거, 계산 집약적 문제도 가능)
        
        → 보험 사례: "이 의료기관의 청구가 정상적일까?"로 재구성하여 사기 확률 예측
        
    - 데이터 분포 일관성: 본 적 없는 데이터가 훈련 데이터와 **유사한 분포**를 가져야 함
        
        → 과거 사기 패턴으로 훈련된 모델이 새로운 사기 수법에도 어느 정도 적용 가능해야 함
        
        | **핵심 조건** | **정의** | **스터디 예시** |
        | --- | --- | --- |
        | **학습 능력** | 시스템이 명시적 규칙이 아닌 데이터로부터 관계를 스스로 학습할 수 있어야 함 | 의료기관 청구 패턴을 입력으로 받아 사기 가능성을 예측하는 모델 학습 |
        | **복잡한 패턴** | 패턴이 존재하며, 규칙 기반 시스템 이상의 복잡도를 지녀야 ML 적용이 유의미 (패턴이 없는 무작위 현상은 부적합함) | 둔갑치료, 영수증 쪼개기 등 다양한 사기 유형의 복합적 패턴 탐지 |
        | **데이터 가용성** | 학습에 사용할 수 있는 데이터가 있거나 수집 가능해야 함 | 장기보상팀의 수동 조사 결과를 축적하여 사기 패턴 데이터베이스 구축 |
        | **예측 문제** | 문제가 예측 형태로 재구성 가능해야 ML 적용이 유효함 (미래, 현재, 계산 문제 모두 포함) | "이 의료기관의 청구가 정상적일까?"를 사기 확률 예측으로 재구성 |
        | **데이터 분포 일관성** | 본 적 없는 데이터가 훈련 데이터와 유사한 분포를 가져야 함 | 과거 사기 패턴으로 훈련된 모델이 새로운 사기 수법에도 어느 정도 적용 가능해야 함 |
    
- ML 솔루션 극대화를 위한 4가지 추가 조건 💸
    - 반복성: 작업이 반복적일수록 대규모 데이터 확보에 유리
        
        → 보험 사례: 매일 수천 건의 의료비 청구 건을 처리하는 반복적 업무 특성에 적합
        
    - 낮은 오류 비용: 잘못된 예측의 비용이 낮거나, 평균적인 이익이 손실을 초과해야 함
        
        → 후보군 제시 방식을 통해 최종 판단은 전문가가 수행함으로써 False Positive/Negative로 인한 오류 비용을 관리
        
    - 대규모 적용: 대량 예측이나 연속 예측처럼 대규모로 적용될 때 비용 효율이 높음
    - 패턴의 지속적 변화: 패턴이 지속적으로 변할 때 ML이 하드코딩 솔루션보다 우수

- ML을 사용해서는 안 되는 경우 🚫
    - 비윤리적인 경우 (예: 자동 채점기 편향 사례)
    - 단순한 솔루션으로 충분한 경우 (규칙 기반, 통계 등으로 해결 가능)
    - 비용 효율적이지 않은 경우 (초기 투자 대비 기대 효과가 낮은 경우)

**1.1.1 머신러닝 유스 케이스**

ML 애플리케이션은 소비자용과 기업용으로 나뉘며, 기업용 ML은 엄격한 정확도와 명확한 ROI를 요구하는 것이 특징

| 주체 | **특징** | **주요 예시** |
| --- | --- | --- |
| **소비자** | 높은 레이턴시 요구사항, 사용자 경험 중심 | 검색, 추천, 기계 번역, 생체 인증 |
| **기업** | 높은 정확도 요구사항, 비용 절감, ROI 증대가 목표 | 이상 거래 탐지, 가격/수요 최적화, 고객 이탈 예측 |

### 1.2 머신러닝 시스템 이해하기 (p. 45)

**1.2.1 연구용 머신러닝 vs 프로덕션용 머신러닝**

| 특징 | 연구용 ML | 프로덕션용 ML |
| --- | --- | --- |
| 최우선 목표 | 정확도 달성 | 비즈니스 임팩트 및 안정성 확보 |
| 주요 Trade-off | 복잡한 모델 vs 시스템 복잡성 | 고성능 vs. 낮은 레이턴시 |
| 성능 측정 | 평균 지표 (Mean) | 백분위수 (Percentiles) 측정 방식 |
| 데이터 환경 | 고정된 데이터 | 끊임없이 변하는 프로덕션 데이터 |
| 추가 고려 | 단순 성능 측정 | 공정성, 해석 가능성, 예산 제약 |

**1.2.2 머신러닝 시스템 vs 전통적인 소프트웨어**

| **구분** | **전통 소프트웨어 (SWE)** | **ML 시스템** |
| --- | --- | --- |
| 핵심 요소 | 코드 중심 | 코드, 데이터, 모델, 인프라의 복합체 |
| 변화 요인 | 코드 변경에 따른 결과 변화 | 데이터 변경에 따른 결과 변화 (비결정성) |
| 관리 대상 | 코드 버전/테스트 | 데이터 버전, 모델 버전, 파이프라인 관리 |
| 결론 | 상대적으로 예측 가능한 단일 관리 대상 | 높은 복잡도의 엔드 투 엔드 시스템 |

### 1.3 정리 (p. 59)

- ML 유스케이스의 주요 가치는 내부 자동화나 처리 비용 절감과 같은 기업 내부에서 발생
- ML이 항상 최선의 답은 아님. 단순한 문제는 규칙 기반 솔루션이 더 빠르고 안정적
- 연구용 ML과 프로덕션 ML은 완전히 다름. 프로덕션에서는 레이턴시, 비용, 안정성이 정확도보다 더 중요
- ML 시스템은 모델 하나가 아닌, 데이터, 배포, 모니터링, 인프라를 모두 포함하는 전체 시스템
